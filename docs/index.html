How it scales

Four optimizations make kernel-scale search practical:

Mmap symbol table
935K symbols loaded via memory-mapped I/O — zero deserialization
Compact chunk metadata
Content stripped from persistence, retrieved from Tantivy on demand (11× smaller)
Lazy trigram loading
Trigram indexes loaded only when Exact strategy or grep_code is used
Content-free trigram index
Posting lists only, no duplicated chunk content (32% smaller on disk)
