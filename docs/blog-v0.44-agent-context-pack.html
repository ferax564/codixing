v0.44.0: agent context packs and runtime MCP profiles

Codixing v0.44.0 turns a set of powerful repository tools into a more usable agent workflow. The release adds agent_context_pack, a stable JSON context compiler for large-repo agents, and lets MCP clients change their active tool profile without killing and restarting the server.

Why this release matters

Codixing already had the hard parts of repository intelligence: Tree-sitter parsing, BM25 and vector retrieval, symbol indexes, dependency graphs, impact analysis, MCP, CLI, LSP, HTTP, and daemon mode. The next problem was making that intelligence easier for agents to use consistently.

An agent should not have to guess whether it needs search, usages, impact, examples, repo maps, docs, or test discovery before it can make progress. It should be able to ask Codixing for the context that matters for the task, under a budget, with stable handles that can be expanded when needed.

The design goal: move Codixing from "many useful tools" toward a repository control plane that assembles focused, safe, measurable context for AI software agents.

agent_context_pack

The new agent_context_pack entry point is the first version of Codixing's context compiler. Given a task, mode, and token budget, it returns a structured JSON package instead of forcing the client to compose a dozen separate searches by hand.

codixing agent-context-pack "add runtime MCP profile switching" --mode edit --token-budget 6000

The same capability is available through MCP as agent_context_pack, so editor agents can ask for one task-local context bundle before reading or editing files.

{
  "task_summary": "Add runtime MCP profile switching",
  "repo_orientation": [
    { "path": "crates/mcp/src/server.rs", "why": "MCP request dispatch and session state" }
  ],
  "must_read": [
    { "id": "E1", "path": "crates/mcp/src/tools/mod.rs", "kind": "tool registry" }
  ],
  "likely_tests": [],
  "risks": [
    "Changing the profile can alter tools/list output for connected MCP clients."
  ],
  "recommended_next_tools": [
    "read_file(E1)",
    "impact(...)"
  ]
}

Stable JSON for agents

The important part is not only the new command. It is the shape of the response. agent_context_pack returns stable sections that agents can reason over without scraping prose:

Field What it gives the agent
repo_orientation Ranked files and symbols that explain where the task lives.
must_read Evidence handles for focused file ranges the agent should inspect first.
related_symbols Definitions and symbols that may be affected by the change.
likely_tests Test files or commands that are likely to cover the affected area.
risks Behavioral, compatibility, and workflow risks to check before editing.
recommended_next_tools The next Codixing tools that should deepen the investigation.

The first supported modes are aimed at real agent workflows: locate, understand, edit, review, test, migrate, and incident. They let clients ask for different context shapes without learning Codixing's full internal routing model.

Runtime MCP profile switching

The other visible improvement fixes a workflow problem: changing the MCP tool profile used to require killing the MCP process and starting it again with different flags. That was painful in editors, especially when an agent started in a safer profile and later needed additional tools for an approved edit.

v0.44.0 adds MCP tools for checking and changing the active profile on the current connection:

get_mcp_profile
set_mcp_profile

After a successful profile change, Codixing emits notifications/tools/list_changed, so MCP clients can refresh their available tools without dropping the session. That keeps read-first workflows practical: start narrow, widen when needed, and avoid a server restart.

What improved for agents

  • Fewer orchestration guesses. Agents can ask for a single context pack instead of guessing the right sequence of search, symbol, usage, impact, and docs calls.
  • Better budget control. Context packs are designed around token budgets and expandable evidence handles.
  • Safer defaults that stay usable. Runtime profile switching makes profile gating less frustrating in real editor sessions.
  • A larger but cleaner MCP surface. The catalog now exposes 70 tools, with profile-aware discovery and a high-level context entry point.
  • Better release coverage. The release ships with 1261 tests and verified Linux, macOS, and Windows binaries.
