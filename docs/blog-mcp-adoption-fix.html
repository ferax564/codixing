Why this benchmark matters

When an AI agent explores code, it pays for every tool call: ~2 seconds of API latency plus hundreds of context tokens consumed. grep returns raw matching lines the LLM must parse. A symbol lookup typically requires 2–4 chained calls: grep to find the file, cat to read it, grep again to find callers.

Codixing replaces that chain with a single structured call. The question is: how much does that actually save, and does the output quality hold up across real codebases?

We measured three things across every task:

  • Tool calls — how many LLM round-trips the agent needs
  • Context tokens — how much output the LLM must parse
  • Structural recall — what percentage of ground-truth items the agent actually found

The grep agent consistently missed files in cross-package dependencies and transitive import chains. These are precisely the cases where grep needs 5+ chained calls to trace, while Codixing's dependency graph resolves them in one.

What grep missed

Specific files the grep agent failed to find across repeated runs:

  • Complexity: createConfigIO, sanitizeChatHistoryMessage — functions with high cyclomatic complexity that don't contain obvious keywords
  • Transitive impact: extensions/telegram/src/bot-native-commands.ts, src/agents/subagent-control.ts, src/auto-reply/reply/agent-runner-payloads.ts — transitively affected files that don't directly import the changed module
  • Blast radius: 5+ extension channel files (discord/shared.ts, imessage/channel.ts, etc.) that implement a shared interface but don't grep-match the type name in an obvious way

When grep is fine

Not everything needs Codixing:

  • Exact keyword search — grep -rn 'fn foo' is fast and precise
  • Reading a known file — cat file.rs needs no index
  • Small codebases — under 20 files, manual navigation is fine
  • Simple single-pattern matching — one grep call, no chaining needed

The crossover point is around 3 chained tool calls. If the task needs grep → read → grep again, Codixing saves time. If it's a single grep, use grep.

Setup

# Plugin (recommended — includes 3 slash commands)
claude plugin marketplace add ferax564/codixing

# MCP only
claude mcp add codixing -- npx -y codixing-mcp --root . --medium --no-daemon-fork

Use --medium to expose 17 core tools directly (no discovery overhead) and --no-daemon-fork to prevent stale socket issues. All 54 tools remain callable.

Methodology

All benchmarks used BM25-only indexing (no embeddings, no GPU). Indexing throughput: ~1,060 files/second. The 6-repo benchmark indexed 9,493 files with 55,869 symbols in 8.8 seconds total.

For the agent benchmark, we used the Claude Agent SDK to run identical prompts in "vanilla" mode (Grep + Glob + Read only) and "codixing" mode (Codixing MCP tools available). 3 runs per task per condition, with statistical significance testing (95% CI, non-overlapping).
